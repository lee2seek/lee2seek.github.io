激活函数zero-center解释

发表于 2018-05-18

最近一直在看cs231-winter 深度学习的课程，其中在讲到激活函数一节，说道sigmoid函数不是zero center的，一直不太理解，今天自己尝试推导一下公式，来验证一下。

首先提出一个问题：如果激活函数不是zero center，将会造成什么问题？
如果激活函数不是zero center，那么某一个神经元上的所有权重的梯度将会是全部为正或者全部为负。下面进行证明

激活函数是sigmoid

首先sigmoid的输出全部大于0，其次sigmoid的导数全部大于0.针对公式1.1,我们分析一下代价函数对于权重的正负号是什么情况。因为${\frac {\partial a_1^{(3)}} {\partial z_1^{(3)}}}$表示的是sigmoid的导数，所以大于0；而${\frac {\partial z_1^{(3)}} {\partial w_{11}^{(3)}}} = a_1^{(2)}$是sigmoid的输出也是大于0，所以$
\frac {\partial E} {\partial w_{11}^{(3)}}$的正负号跟${\frac {\partial E} {\partial a_1^{(3)}}}$相同，同理我们可以分析公式1.2，代价函数对于$w_{12}^{(3)}$的导数也跟${\frac {\partial E} {\partial a_1^{(3)}}}$相同，所以代价函数对于连接到同一个神经元的权重的导数的符号全部相同，要么全部大于0，要么全部小于0。所以当更新这些权重时，将会同时往同一个方向前进，这样子权重的学习将会变慢。所以说sigmoid激活函数不是zero-center的

激活函数是tanh

公式1.1和公式1.2前两项相同，第三项不同。对于tanh激活函数，输出值有正有负，所以$a_1^{(2)}$和$a_2^{(2)}$的正负号不同，所以代价函数对于连接到同一个神经元的权重的导数的符号不能保证相同，所以能够保证$w_{11}^{(3)}$和$w_{12}^{(3)}$的更新可以朝不同的方向更新。

激活函数

发表于 2018-05-06 | 分类于 深度学习

激活函数

深度网络中的激活函数通常会使用ReLU函数，但是这一函数并不具有普适性。

ReLU激活函数

$$g(z)=max{0,z}$$
ReLU通常作用于经过放射变换(就是线性映射)后的之上：
$$h=g(W^Tx+b)$$
ReLU函数有一个缺点，不能采用梯度方法来学习使得激活函数为0的样本.针对这个缺点，有三种扩展的ReLU激活函数，这三个扩展都是针对$z_i<0$时，使用一个非零的斜率$\alpha_i$:
$$h_i=g(z,\alpha)_i=max(0,z_i)+\alpha_imin(0,z_i)$$

  1. absolute value rectification(绝对值整流)
    取$\alpha_i=-1$,即可以得到 $g(z)=|z|$. 这种相对于其他两种使用的相对较少
  2. Leaky ReLU
    取$\alpha_i=0.01$
  3. parametric ReLU
    将$\alpha_i$作为学习的参数
maxout unit

maxout 激活函数也是ReLU的一种扩展，没有理解maxout激活函数

Logistic Sigmoid 和 Hyperbolic Tangent

Logistic Sigmoid激活函数

$$g(z)=\sigma(z)=1/(1+e^{-x})$$
Sigmoid激活函数当输入很大或者很小时，输出基本上不会有什么区别。当输入在0值的附近时，激活函数的输出才对输入比较敏感。这里的敏感我个人觉得是指当输入有变化时，激活函数的输出的变化程度。比如当的输入从100增加到101，激活函数的输出基本上是相同的。因为这个原因，在前馈网络中不推荐使用Sigmoid激活函数

双曲正切函数

$$g(z)=tanh(z)=(e^x-e^{-x})/(e^x+e^{-x})$$
一般来说tanh(z)函数比sigmoid函数表现更好
注意：$tanh(z)=2\sigma(2z) - 1$

其他激活函数

这里的激活函数用的相对较少。

  1. RBF Unit(径向基函数)
  2. Softplus
    $$g(a)=\zeta(a)=log(1+e^a)$$
  3. Hard tanh
    $$g(a)=max(-1,min(1,a))$$

kubernetes入门

发表于 2018-05-04

kubernetes可以部署,调度和节点集群间扩展。实际上,Kubernetes就是一个部署文件。

  • 可以自动化部署容器以及容器复制
  • 随时扩展容器规模
  • 提供容器间的负载均衡
  • 如果容器失效，可以马上进行替换

关键概念

  1. Kubernetes Master
  2. Node
  3. Pod
  4. Replication Controller
  5. Service
  6. Labels
  7. Container
  8. API Server

Kubernetes Master

Master 拥有API Server及Replication Controller.Replication Controller用来创建和复制Pod。API Server提供REST操作，能够跟集群中的组件进行交互。

Node

node是物理或虚拟机器，作为Kubernetes Worker，通常也被称为Minion。包含以下三个关键组件：

  • Kubelet：主节点代理
  • Kube-proxy：Service使用其来将链接路由到对应Pod上
  • Docker：K8s使用的容器技术来创建容器

Pod

Pod是一个kubernetes的抽象，代表了一组应用容器。pod是工作在一个私有的，隔离的网络中的，集群内pod之间是可以相互访问。所以要从外部访问pod，需要打开代理 kubectl proxy.
pod是有生命周期的，当node节点挂了，那么其上的pod也就丢失了。这里其实牵涉到一个问题，假设pod节点丢失了，前端是不应该感受到的，k8s自动帮你创建丢失的pod，而这个就是Replication Controller做的工作

假设我们需要查看某一个pod上面的应用，那么我们应该怎么办呢？

  1. 首先打开代理，kubectl proxy
  2. 获取pod的名称
  3. 使用curl命令获取莹莹的输出

Replication Controller

Replication Controller确保任意时间都有指定数量的Pod副本运行。

Service

A Kubernetes Service is an abstraction layer which defines a logical set of Pods and enables external traffic exposure, load balancing and service discovery for those Pods.

线性代数知识

发表于 2018-02-12

这里只记录我认为需要复习的一些知识

1.1 标量、向量、矩阵和张量

  • 标量: 就是一个单独的数
  • 向量: 一列有序排列的数
  • 矩阵：一个二维数组
  • 张量(tensor): 坐标超过两维的数组

1.2 线性相关和生成子空间

  • 生成子空间：原始向量线性组合后能抵达的点的集合
  • 线性无关：一组向量中的任意一个向量都不能表示成其他向量的线性组合
  • 奇异的方阵：列向量线性相关的方阵

1.3 范数

机器学习中经常使用范数(norm)衡量向量大小,
常用的范数有L2,L1,以及最大范数。

1.4 特殊类型的矩阵和向量

  1. 对角矩阵：对角线上含有非零元素，其他位置的元素都是零

    对角矩阵的逆存在，当且仅当对角元素都是非零值，其逆矩阵就是将对应的对角线元素取倒数

  2. 对称矩阵：矩阵的转置和自己相等的矩阵

  3. 标准正交：向量不仅互相正交，并且范数都是1
  4. 正交矩阵：行向量和列向量分别标准正交的方阵

1.5 特征分解

用来发现矩阵的一些内在性质

特征分解是将矩阵分解为一组特征向量和特征值
$$A\upsilon=\lambda\upsilon$$
特征向量：方阵$A$的特征向量是指与$A$相乘后相当于对该向量进行缩放的非零向量$\upsilon$
。如上述公式中，$\upsilon$是特征向量，$\lambda$是特征向量对应的特征值
。同样也可以定义左特征向量$\upsilon^{\rm T}A = \lambda\upsilon^{\rm T}$。但是一般更关注右特征向量。

deep learning read note

发表于 2018-02-09

深度学习的阶段

  1. 控制论

    控制论主要是研究设置一组权重来学习如何用输入的线性表示输出，但是线性模型无法解决异或(XOR)问题

  2. 联结主义

    当网络将大量简单的计算单元连接在一起时可以实现智能行为，反向传播算法也是这一时期。当动物的许多神经元一起工作时会变得聪明，单独神经元或小集合的神经元不是特别有用

  3. 深度学习

    深度学习并不是在尝试模拟大脑，现代的深度学习从很多领域来获取灵感，特别是数学中的线性代数，概率论，信息论与数值优化

推动深度学习发展的因素

  1. 越来越多的数据

    这些年研究的数据集也是越来越大，深度学习在大数据量上的表现更好，精度也更高

  2. 计算机软件以及硬件的发展

    每神经元的连接数目以及神经元的总数目对于动物的智力有很重要的影响。模型的规模的不断增加时深度学习历史中最重要的趋势之一

  3. 与日剧增的精度、复杂度对现实世界的冲击

    现在深度学习在很多任务上取得了很高的精度
